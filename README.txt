محرك معرفة RHEL العربي — المرحلة الرابعة
Linux Doctor V1

الفكرة:
Linux Doctor هو معالج تشخيص تفاعلي يعمل داخل المتصفح. لا ينفذ أي أمر على الجهاز، بل يعطي أمر فحص، ثم يغيّر مسار التشخيص بناءً على النتيجة التي يختارها المستخدم.

المزايا الجديدة:
- 9 مسارات تشخيص رئيسية.
- 111 عقدة قرار وفحص.
- 65 نتيجة تشخيص نهائية.
- حفظ الجلسة تلقائياً وإمكانية متابعتها لاحقاً.
- تعبئة المتغيرات داخل الأوامر مثل SERVICE وHOST وPORT وPATH.
- رجوع إلى الخطوة السابقة وإعادة التشخيص.
- نسخ كل أمر بشكل مستقل.
- إنشاء تقرير تشخيص نصي ونسخه أو تنزيله.
- الانتقال من النتيجة إلى صفحة المعرفة المرتبطة.
- يعمل محلياً دون API أو إرسال بيانات المستخدم.

مسارات Linux Doctor:
1. خدمة لا تعمل أو تظهر Failed.
2. تعذر الاتصال عبر SSH.
3. القرص ممتلئ أو No space left.
4. فشل تثبيت برنامج أو حزمة بواسطة DNF.
5. انقطاع الشبكة أو DNS أو الإنترنت.
6. بطء النظام أو ارتفاع الحمل.
7. Permission denied والصلاحيات.
8. خدمة أو منفذ لا يمكن الوصول إليه.
9. تشخيص رفض SELinux.

الملفات الجديدة:
- doctor-data.json
  أشجار القرار والأسئلة والفحوصات والنتائج والإجراءات.

- doctor-engine.js
  محرك الجلسات والتفرعات والمتغيرات والتقارير.

- test-doctor.js
  يتحقق من سلامة جميع الروابط بين عقد التشخيص والمراجع.

- DOCTOR_TEST_REPORT.txt
  نتيجة آخر فحص لبيانات Linux Doctor.

الرفع إلى GitHub:
ارفع جميع الملفات الموجودة في الحزمة إلى جذر المستودع واستبدل الملفات الحالية. لا تغيّر إعدادات Cloudflare Pages.

اختبار الملفات محلياً:
node --check app.js
node --check intent-engine.js
node --check doctor-engine.js
node test-intents.js
node test-doctor.js

تشغيل الموقع محلياً:
python -m http.server 8000

http://localhost:8000

تطوير الطبيب:
1. أضف أو عدّل مساراً في doctor-data.json.
2. تأكد أن كل choice.next يشير إلى عقدة موجودة.
3. شغّل node test-doctor.js.
4. ارفع doctor-data.json إلى GitHub.
